# 데이터시트 프리뷰 패널 재설계

## Context
현재 프리뷰 패널 `h-[120px]`은 분류 4개 필드를 한 줄로 압축하고, quantity/unit 미표시, specification/notes 양자택일 등 정보를 지나치게 생략하고 있음. 프리뷰는 데이터 그리드보다 **더 자세한** 정보를 제공해야 함. 품의서 페이지의 `h-[340px]` 히어로 카드 패턴을 참고하여 전면 재설계.

## 레이아웃 계획

### 높이: `h-[300px]` (서브헤더 40px + 본문 260px)

## 수정 파일

### 1. `DatasheetPreviewPanel.tsx` — 전면 재작성

**서브헤더 (h-[36px])**:
- doc_number + StatusBadge + 유형칩 + date + 총액(bold) + 상세링크

**좌측 정보 (w-[440px])**:
- `<table>` 4열 (label-value-label-value), Procurement 페이지 패턴 참고
- 행 1: 고객사 / 설비명
- 행 2: 거래처 / 담당(이름+직급)
- 구분선
- 행 3: 유니트 / 대분류
- 행 4: 중분류 / 소분류
- 구분선
- 행 5: 규격 (colspan=3)
- 행 6: 수량+단위 / 팀
- 구분선 (노트가 있을 때만)
- 행 7: 노트 (flex-1, 남는 공간)

**우측 금액 카드 (flex-1)**:
- 견적가액: 라벨 + 금액 + 수평 바 (bg-muted-foreground/20, 기준 100%)
- 공급가액: 라벨 + 금액 + 수평 바 (bg-primary, estimate 대비 비율)
- 세액: 라벨 + 금액 (바 없음)
- 구분선
- 총액: font-semibold 강조
- 차액: emerald(절감) / red(초과) + 비율
- 카드번호: 영수증만 (font-mono)

### 2. `DatasheetClient.tsx` — 최소 수정

- hero card 컨테이너의 프리뷰 영역만 변경
- `onClose` prop 전달은 불필요 (항상 첫 행 자동선택, 닫기 없음)

## 검증

1. `tsc --noEmit` — 타입 체크

3. 키보드 ↑↓로 항목 전환 시 프리뷰 즉시 반영 확인
