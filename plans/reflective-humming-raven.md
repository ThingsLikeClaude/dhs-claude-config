# 사이드바 다크모드 활성 페이지 하이라이트 가시성 개선

## Context

다크모드에서 사이드바의 선택된(활성) 페이지 하이라이트가 `dark:bg-white/10` (10% 불투명도 흰색)으로 설정되어 있어, `dark:bg-zinc-950/60` 사이드바 배경 위에서 거의 구분이 안 되는 문제.

## 수정 대상

**파일**: `frontend/src/components/layout/Sidebar.tsx`

활성 상태 스타일이 동일하게 적용된 **3곳**:
1. **Line 193** — 대시보드 링크
2. **Line 231** — 일반 네비 항목
3. **Line 271** — 마스터 드롭다운 항목

현재 활성 클래스 (다크모드 관련):
```
dark:bg-white/10 dark:ring-white/[0.06]
```

## 수정 내용

`dark:bg-white/10` → `dark:bg-white/[0.15]` 로 변경하여 배경 불투명도를 높임.
`dark:ring-white/[0.06]` → `dark:ring-white/[0.12]` 로 변경하여 테두리도 더 뚜렷하게.

이 조합은:
- 기존 glassmorphism 디자인 톤을 유지하면서
- 활성 항목이 비활성 항목과 명확히 구분됨
- 브랜드 컬러 `text-primary`와 함께 시너지

> 만약 `0.15`로도 부족하면 `0.20`까지 올릴 수 있음. 과하면 라이트모드와의 균형이 깨지므로 `0.15`부터 시작.

## 수정 범위

| 위치 | 변경 |
|------|------|
| Line 193 | `dark:bg-white/10` → `dark:bg-white/[0.15]`, `dark:ring-white/[0.06]` → `dark:ring-white/[0.12]` |
| Line 231 | 동일 |
| Line 271 | 동일 |

총 1개 파일, 3곳 수정. 다른 파일 변경 없음.

## 검증

1. `pnpm build` 성공 확인
2. 브라우저에서 다크모드 사이드바 확인 — 활성 페이지가 비활성과 시각적으로 구분되는지 확인
